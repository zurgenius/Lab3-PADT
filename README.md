# Lab 3: АТД последовательностей и интерполяция сплайнами

Проект реализует набор абстрактных типов данных для работы с последовательностями и
использует их для построения кусочно-линейной и кубической интерполяции. Есть
консольное меню, модульные тесты и графический просмотрщик на GLFW, Dear ImGui и ImPlot.

## Возможности

- `DynamicArray<T>`: динамический массив с изменением размера, доступом по индексу и
  базовыми операциями вставки/удаления.
- `LinkedList<T>`: связный список с операциями добавления, вставки, удаления и обхода.
- `Sequence<T>`: общий интерфейс последовательностей.
- `ArraySequence<T>` и `ListSequence<T>`: реализации `Sequence<T>` на базе
  `DynamicArray<T>` и `LinkedList<T>`.
- Mutable- и immutable-варианты последовательностей.
- `Option<T>` для безопасного возврата значения без исключений при обычном отсутствии
  результата.
- `IEnumerator<T>` для универсального обхода структур данных.
- Функции высшего порядка для последовательностей: `map`, `where`, `reduce`, `slice`,
  `concat`.
- Обобщенный интерфейс `Interpolator<T>` для интерполяторов.
- `LinearSplineInterpolator<T>` для кусочно-линейной интерполяции.
- `CubicSplineInterpolator<T>` для натурального кубического сплайна.
- `Field` concept для ограничения числовых типов, пригодных для интерполяции.
- Представление интерполяции как набора `Function<T>`-сегментов.
- Графический просмотрщик сплайнов с режимами:
  - пользовательские точки;
  - сравнение `sin(x)` и интерполяции в одной системе координат;
  - переключение между линейным и кубическим интерполятором;
  - отображение узлов и аналитического кусочного вида функции.

## Структура проекта

- `include/` — публичные заголовки.
- `include/detail/` — шаблонные реализации `*.tpp`.
- `src/` — исходники приложения, консольное меню и GUI.
- `tests/` — модульные тесты GoogleTest.
- `CMakeLists.txt` — CMake-конфигурация проекта.
- `Makefile` — короткие команды для сборки, тестов, форматирования и lint-сборки.

## Сборка и запуск

Проект использует C++20 и CMake. Зависимости GoogleTest, GLFW, Dear ImGui и ImPlot
подключаются через `FetchContent`.

```bash
make program
```

Команда конфигурирует проект, собирает приложение и кладет исполняемый файл в
`./program`.

Запуск:

```bash
make run
```

Ручная сборка через CMake:

```bash
cmake -S . -B build -DWITH_SPLINE_VIEWER=ON
cmake --build build
```

Важно: не удаляйте `build/_deps`, если не нужно полностью сбросить зависимости.
В этой директории CMake кеширует загруженные библиотеки.

## Тесты

```bash
make tests
```

Команда собирает тестовый бинарник и запускает `ctest`.

Ручной запуск:

```bash
cmake --build build --target lab2_tests
ctest --test-dir build --output-on-failure
```

## Форматирование и lint

```bash
make format
make lint
```

`make format` запускает `clang-format` для файлов в `include/`, `src/` и `tests/`.
`make lint` собирает проект в отдельной директории `build-lint` с `-Werror` и запускает
тесты.

## Графический режим

GUI включен по умолчанию опцией CMake `WITH_SPLINE_VIEWER=ON`. В приложении нужно
выбрать пункт меню запуска просмотрщика сплайнов.

В режиме пользовательских точек можно добавлять точки через поля ввода или кликом по
графику. Для корректной интерполяции значения `x` должны быть строго возрастающими.

В режиме сравнения строятся две кривые в одной ДПСК:

- исходная функция `sin(x)`;
- выбранная интерполяция по узлам с шагом `0.5` по оси `x`.

Разница между кривыми подсвечивается полупрозрачной заливкой, узлы интерполяции
отображаются маркерами. Начальный масштаб графика сфокусирован на участке, где различие
между `sin(x)` и интерполяцией заметнее; при необходимости график можно масштабировать и
двигать мышью.
